COMP229 - Final Project - Part 1: Requirements

Note: Students often ask how detailed they need to be. There is guidance below, but the best way to think about it is not as an assignment but as actual requirements that you will use to help everyone on your team communicate effectively. Any minute you spend doing requirements or wireframes now will save you 2-3 minutes of coding. So, detail is good for you!

Note 2: Another key point is that I want to ensure the project is the right level of challenge for you. It should push you to learn and grow but still be achievable within the given timeline and with your current skills. Choosing features that are challenging but doable will help you get the most out of this project and feel proud of what you’ve accomplished.

For this first part of your group project, your task is to define the requirements for your MERN application, which includes gathering the necessary details to fully scope out your project. You will need to document both the technical aspects and user experiences you aim to provide. The requirements section should cover the following components:

Project Requirements

  • Technology Stack: You must use React, Node, and a database of your choice.

  • Challenge Level: Select a project scope that will be challenging yet achievable for you and your team.

  • Portfolio Focus: Build your project in a way that makes it suitable for your professional portfolio. If you use it in your portfolio, make sure to outline your specific contributions.

  • Interest Driven: Choose something that you find interesting and are passionate about! If you find it interesting, you’ll stay motivated to keep coding, which is the ultimate goal!

Project Breakdown

Functional Requirements (30%)

List all the features that you plan to build here. Make sure to include both the required features and any stretch goals you want to attempt. Being detailed about all the features will help ensure you understand the full scope of your project and will guide your development later on.

Label all your features as either Required or Stretch Goal.

  • Required Features: These are the core functionalities that must be completed to ensure the application is fully functional. Think of these as the essential features that provide the core functionality of the application.

  • Stretch Goals: These are additional features that would enhance the application but are not essential for it to be functional. Attempt these features if you want to push your learning and if time permits.

User Stories and Use Cases (30%)

  • Write user stories that describe the features from the user perspective.
  • Develop use cases that outline the steps users will take to interact with the application and what they can expect as a result.

Wireframes (40%)

  • Create wireframes for all major screens of your application, ensuring that they show the layout, navigation, and key interactions for each screen. These should represent the visual structure of the app and help guide your development.
  • Example: Here is a good read on wireframes with some examples
  • Tools like Figma, Balsamiq, or even an image editor like Paint can be used to create wireframes.

Examples

Todo List App Example

  • Functional Requirement (Required): The application must allow users to create, read, update, and delete (CRUD) their tasks. Each task must include a title and due date.

  • Functional Requirement (Stretch Goal): Users should be able to categorize tasks by labels or tags and filter the task list based on these categories.

  • User Stories:

    • “As a user, I want to be able to add a new task so that I can keep track of my to-dos.”
    • “As a user, I want to mark tasks as complete so that I can easily see what I have accomplished.”
    • “As a user, I want to filter my tasks by category so that I can focus on specific areas of my to-do list.”

  • Use Case:

    • Title: Adding a New Task
    • Actor: User
    • Description: The user opens the app, navigates to the ‘Add Task’ screen, enters a task title and due date, and then clicks the ‘Save’ button. The system confirms the task has been added successfully and displays it in the task list.
